# คู่มือการ Sync โค้ดระหว่างบ้าน (main) และที่ทำงาน (dev-work)

เนื่องจากเราตั้งค่าให้ไฟล์ `config.php` (ที่เก็บรหัสผ่าน Database) **ไม่ถูกนำขึ้น Git** การดึงโค้ดข้ามเครื่องจึงสามารถทำได้อย่างปลอดภัย โดย Database ของแต่ละเครื่องจะไม่พังและไม่ถูกทับแน่นอนครับ

หลักการทำงานคือ:
1. ทำงานเสร็จ ต้อง **Commit** และ **Push** โค้ดขึ้นไปเก็บบน GitHub ก่อนกลับ
2. พอไปถึงอีกเครื่อง ให้ **Pull/Merge** โค้ดจากของเครื่องเดิมลงมา

---

## 🏢 สถานการณ์ที่ 1: มาทำงานที่บริษัท (ใช้ dev-work) อยากอัปเดตโค้ดที่เพิ่งทำมาจากบ้าน (main)

เมื่อวานทำโค้ดที่บ้านเสร็จ (เซฟลง branch `main` บน GitHub แล้ว) วันนี้มาถึงที่ทำงาน เปิดคอมบริษัทขึ้นมา:

1. เช็คให้ชัวร์ว่าตอนนี้เราอยู่ branch ของที่ทำงาน
   ```bash
   git checkout dev-work
   ```
2. ดึงโค้ดล่าสุดจาก GitHub (อัปเดตข้อมูล)
   ```bash
   git fetch origin
   ```
3. นำโค้ดหน้าเว็บจาก branch `main` ของที่บ้าน มาทับ/อัปเดต ใส่ในเครื่องนี้
   ```bash
   git merge origin/main
   ```
*(ถ้าเจอหน้าจอให้ใส่ข้อความ Merge ยืนยัน สามารถพิมพ์ `:wq` แล้วกด Enter เพื่อผ่านไปได้เลยครับ)*

---

## 🏠 สถานการณ์ที่ 2: กลับมาทำที่บ้าน (ใช้ main) อยากดึงโค้ดที่เพิ่งทำที่ทำงาน (dev-work) มาทำต่อ

ตอนเย็นก่อนเลิกงาน คุณเซฟงานที่บริษัท (Commit + Push โค้ดของ `dev-work` ขึ้น GitHub แล้ว) พอกลับมาถึงบ้าน:

1. เช็คให้ชัวร์ว่าคอมที่บ้านอยู่ branch ของบ้าน
   ```bash
   git checkout main
   ```
2. ดึงข้อมูลล่าสุดจาก GitHub
   ```bash
   git fetch origin
   ```
3. นำโค้ดที่เพิ่งทำที่ทำงานมาอัปเดตลงเครื่องบ้าน
   ```bash
   git merge origin/dev-work
   ```

---

## 🎯 กฎเหล็กที่สำคัญที่สุด!
- **ห้าม** ใช้คำสั่ง `git reset --hard` เด็ดขาดถ้าไม่จำเป็น เพราะมันจะลบ "งานที่เพิ่งทำค้างไว้แล้วยังไม่ได้เซฟ (Commit)" ทิ้งไปจนหมดกู้คืนไม่ได้ครับ
- คำสั่ง `git merge` คือพระเอกของเรา มันจะเอา "โค้ดรวมๆ กัน" โดยไม่ไปยุ่งกับไฟล์วิเศษอย่าง `config.php` ของเราครับ
