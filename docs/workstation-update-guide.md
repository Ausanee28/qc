# คู่มืออัปเดตโปรเจกต์บนเครื่องทำงาน (หลัง `git pull`)

เอกสารนี้สรุปขั้นตอนสั้น ๆ สำหรับอัปเดตโปรเจกต์ให้พร้อมใช้งานหลัง pull โค้ดล่าสุด

## ขั้นตอนมาตรฐาน

1. ดึงโค้ดล่าสุด

```bash
git pull
```

2. เคลียร์ cache ของ Laravel (แนะนำทำทุกครั้งหลัง pull)

```bash
php artisan optimize:clear
```

3. รันตามโหมดที่ใช้งาน

- โหมดพัฒนา (Dev)

```bash
npm run dev
```

- โหมด build ไฟล์ใช้งานจริง

```bash
npm run build
```

## ต้องติดตั้งเพิ่มไหม?

- ถ้าเครื่องนี้เคย setup แล้ว: ปกติไม่ต้องติดตั้งอะไรเพิ่ม
- ถ้าเพิ่ง clone ใหม่หรือมี dependency เปลี่ยน: ให้รัน

```bash
composer install
npm install
```

## ต้อง migrate ทุกครั้งไหม?

- ไม่จำเป็นต้อง migrate ทุกครั้ง
- ให้รันเฉพาะตอนมี migration ใหม่

```bash
php artisan migrate
```

## เช็กว่าเว็บพร้อมใช้งาน

1. เปิดหน้าเว็บและลองเปลี่ยนเมนูหลัก 2-3 หน้า
2. ทดสอบหน้า `Receive Job` และ `Execute Test` ว่าโหลดข้อมูลได้ปกติ
3. ถ้าหน้าไม่อัปเดต ให้ hard refresh (`Ctrl+F5`) 1 รอบ
