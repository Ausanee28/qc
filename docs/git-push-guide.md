# สรุปวิธีเอาโค้ดขึ้น GitHub แบบเข้าใจง่ายๆ (Git Workflow)

การเซฟงานและส่งโค้ดขึ้นไปเก็บบน GitHub มักจะมี 3 สเต็ปหลักๆ ที่ต้องทำเรียงกันเสมอครับ เปรียบเสมือนการ "แพ็คของส่งไปรษณีย์" ครับ

---

## สเต็ป 1: หยิบของใส่กล่อง (`git add`)
คำสั่งนี้เหมือนเป็นการเลือกไฟล์ที่เราแก้ไขเสร็จแล้ว เตรียมพร้อมบรรจุลงกล่อง

- ถ้าต้องการหยิบ **"ทุกไฟล์"** ที่เปลี่ยนไปใส่กล่องทั้งหมด (เจอบ่อยสุด)
  ```bash
  git add .
  ```
- ถ้าอยากเลือกหยิบแค่ **"บางไฟล์"** (เช่นแก้หลายไฟล์แต่เพิ่งเสร็จไฟล์เดียว)
  ```bash
  git add index.php
  ```

---

## สเต็ป 2: ปิดเทป แปะป้ายบอกว่าของข้างในคืออะไร (`git commit`)
คำสั่งนี้คือการเซฟแพ็คเกจนั้นไว้ในเครื่องเรา โดยเราต้องตั้งชื่อป้ายแปะกล่องด้วย เพื่อที่วันหลังกลับมาดูจะได้รู้ว่าอัปเดตอะไรไป 

```bash
git commit -m "ใส่ข้อความอธิบายสิ่งที่แก้ตรงนี้"
```
**ตัวอย่างการเขียนสั้นๆ เข้าใจง่าย:**
- `git commit -m "fix: แก้บักปุ่มกดไม่ได้"`
- `git commit -m "feat: เพิ่มหน้ารายงานใหม่"`
- `git commit -m "style: ปรับสีตารางให้สวยขึ้น"`

---

## สเต็ป 3: ส่งกล่องขึ้นรถไปรษณีย์ (ฝากขึ้น GitHub) (`git push`)
ขั้นตอนสุดท้าย เป็นการอัปโหลดกล่องทั้งหมดที่เราเตรียมไว้ (Commit) ตามสายอินเตอร์เน็ตขึ้นไปเก็บบน GitHub

- ถ้านี่เป็น **การ Push ครั้งแรก** ของ Branch ที่เราเพิ่งสร้างใหม่ (Git บนเว็บยังไม่รู้จัก Branch นี้) ต้องพิมพ์เต็มๆ ใบเบิกทางก่อน 1 ครั้ง:
  ```bash
  git push -u origin ชื่อ_branch_ของเรา
  ```
  *(เช่น `git push -u origin dev-work`)*

- แต่ถ้าเคยส่งไปแล้ว หรือนี่คือ Branch ทั่วไปที่ใช้อยู่แล้ว พิมพ์แค่สั้นๆ ได้เลยครับ **(ส่วนใหญ่จะเป็นอันนี้)**
  ```bash
  git push
  ```

---

## 🛠️ สรุปคำสั่ง 3 บรรทัดจบงาน ท่องไว้ใช้ได้ตลอด:

เปิด Terminal (PowerShell/CMD) แล้วพิมพ์ 3 บรรทัดนี้เรียงกันได้เลยครับ

```bash
git add .
git commit -m "อัปเดตงานวันนี้"
git push
```

*(จบปิ๊ง! โค้ดขึ้นไปนอนรอในเว็บไซต์เรียบร้อยแล้วครับ 🎉)*
